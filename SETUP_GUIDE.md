## 📊 Supabaseのセットアップ

### ステップ1: Supabaseプロジェクトの作成

1. [Supabase](https://supabase.com/)にアクセスしてサインイン
2. 「New Project」ボタンをクリック
3. プロジェクト名を入力（例：`kouteihyo-prod`）
4. データベースパスワードを設定（安全な場所に保存してください）
5. リージョンを選択（日本の場合は`Northeast Asia (Tokyo)`推奨）
6. 「Create new project」をクリック

### ステップ2: データベーステーブルの作成

1. プロジェクトダッシュボードで「SQL Editor」をクリック
2. 「New query」をクリック
3. `supabase-setup.sql`ファイルの内容を全てコピー＆ペースト
4. 「Run」ボタンをクリックして実行
5. 成功メッセージが表示されることを確認

### ステップ3: API情報の取得

1. プロジェクトダッシュボードで「Settings」→「API」をクリック
2. 以下の情報をメモ：
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon/public key**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`（長い文字列）

## ⚙️ アプリケーションの設定

### Vercelの環境変数設定

1. [Vercel Dashboard](https://vercel.com/dashboard)にアクセス
2. `kouteihyo`プロジェクトを選択
3. 「Settings」→「Environment Variables」をクリック
4. 以下の環境変数を追加：

| 変数名 | 値 | 説明 |
|--------|-----|------|
| `VITE_SUPABASE_URL` | `https://xxxxx.supabase.co` | SupabaseのProject URL |
| `VITE_SUPABASE_KEY` | `eyJhbGciOi...` | Supabaseのanon key |

5. 「Save」をクリック
6. プロジェクトを再デプロイ（「Deployments」→最新デプロイの「…」→「Redeploy」）

## 🔒 セキュリティ設定

### オプションA: 認証なし（社内イントラネット向け）

現在の設定（`supabase-setup.sql`のデフォルト）では、誰でもアクセス可能です。

**⚠️ 注意**: 
- 社内ネットワークなど、限られた環境での使用を推奨
- 機密情報は入力しないでください
- 定期的なバックアップを推奨

### オプションB: Supabase認証を追加（推奨）

より安全にするには、Supabase Authを実装してください：

1. `supabase-setup.sql`の「オプションB」のコメントを外す
2. アプリケーションにログイン機能を追加（開発が必要）
3. ユーザー登録・管理機能の実装

### オプションC: IP制限

Vercelの設定でIPアドレス制限を追加することも可能です（Pro プラン以上）。

## 🐛 トラブルシューティング

### タスクが表示されない

**原因**: 環境変数が正しく設定されていない

**解決方法**:
1. Vercelの環境変数を再確認
2. 変数名が完全に一致しているか確認（大文字小文字含む）
3. プロジェクトを再デプロイ

### 「RLS policy violation」エラー

**原因**: Row Level Securityのポリシーが正しく設定されていない

**解決方法**:
1. Supabaseの「SQL Editor」を開く
2. 以下のクエリを実行して、ポリシーを確認：
```sql
SELECT * FROM pg_policies WHERE tablename = 'tasks';
```
3. ポリシーが存在しない場合、`supabase-setup.sql`を再実行

### リアルタイム更新が動作しない

**原因**: Replicationが有効になっていない

**解決方法**:
1. Supabaseで「Database」→「Replication」を開く
2. `tasks`テーブルが有効になっているか確認
3. 無効の場合は有効化する

### 削除確認ダイアログが表示されない

**原因**: ブラウザの設定でダイアログがブロックされている

**解決方法**:
1. ブラウザの設定で「ダイアログの表示を許可」を確認
2. 別のブラウザで試す

### モバイルで表示が崩れる

**原因**: ブラウザのキャッシュが古い

**解決方法**:
1. ブラウザのキャッシュをクリア
2. ページをリロード（強制リロード: Ctrl+Shift+R / Cmd+Shift+R）

## 🔄 バックアップとメンテナンス

### 定期バックアップの推奨

1. Supabaseダッシュボードで「Database」→「Backups」を開く
2. 手動バックアップを作成、または自動バックアップを設定
3. 重要なデータは定期的にエクスポート

### データのエクスポート方法

```sql
-- SQL Editorで実行
COPY (SELECT * FROM tasks ORDER BY created_at DESC) 
TO '/tmp/tasks_backup.csv' 
WITH (FORMAT CSV, HEADER);
```

## 📈 パフォーマンス最適化

大量のタスクがある場合（1000件以上）:

1. 古いタスクを定期的にアーカイブ
2. 不要なタスクを削除
3. インデックスのパフォーマンスを確認
